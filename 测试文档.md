# 相机租赁系统测试文档

## 一、系统架构

### 1.1 技术栈
- **前端**: 微信小程序 (WXSS/WXML/JavaScript)
- **后端**: Node.js + Express + MySQL
- **网络**: Cloudflare Tunnel (内网穿透)
- **域名**: sunnycamera.online

### 1.2 项目结构
```
d:\相机租赁\
├── backend\                 # 后端服务
│   ├── config\              # 配置文件
│   │   └── database.js      # 数据库配置
│   ├── routes\              # API路由
│   │   └── api.js           # 主要API接口
│   ├── server.js            # 服务器入口
│   ├── check-database.js    # 数据库检查脚本
│   ├── setup-db.js          # 数据库初始化脚本
│   └── .env                 # 环境变量
├── miniprogram\             # 微信小程序
│   ├── pages\               # 页面文件
│   ├── images\              # 图片资源
│   └── app.json             # 小程序配置
├── cloudflared.exe          # Cloudflare Tunnel客户端
└── test.html                # 测试页面
```

## 五、数据库表结构

### 5.1 cameras (相机表)
```sql
CREATE TABLE cameras (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(255) NOT NULL,
  brand VARCHAR(255) NOT NULL,
  model VARCHAR(255) NOT NULL,
  price DECIMAL(10, 2) NOT NULL,
  description TEXT,
  image_url VARCHAR(255),
  status ENUM('available', 'rented', 'maintenance') DEFAULT 'available',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 5.2 users (用户表)
```sql
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  openid VARCHAR(255) NOT NULL UNIQUE,
  nickname VARCHAR(255),
  avatar VARCHAR(255),
  phone VARCHAR(20),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 5.3 user_sessions (用户会话表)
```sql
CREATE TABLE user_sessions (
  id INT AUTO_INCREMENT PRIMARY KEY,
  user_id INT NOT NULL,
  session_key VARCHAR(255) NOT NULL,
  expires_at TIMESTAMP NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### 5.4 rentals (租赁表)
```sql
CREATE TABLE rentals (
  id INT AUTO_INCREMENT PRIMARY KEY,
  camera_id INT NOT NULL,
  user_name VARCHAR(255) NOT NULL,
  user_phone VARCHAR(20) NOT NULL,
  start_date DATE NOT NULL,
  end_date DATE NOT NULL,
  total_price DECIMAL(10, 2) NOT NULL,
  status ENUM('active', 'completed', 'cancelled') DEFAULT 'active',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (camera_id) REFERENCES cameras(id) ON DELETE CASCADE
);
```

## 六、故障排查

### 6.1 后端服务无法启动

#### 问题1：端口被占用
**症状**: 错误信息 `EADDRINUSE: address already in use :::3000`

**解决**:
```bash
# 查找占用端口的进程
netstat -ano | findstr :3000

# 结束进程 (将 <PID> 替换为实际的进程ID)
taskkill /PID <PID> /F

# 重新启动服务
node server.js
```

#### 问题2：数据库连接失败
**症状**: 错误信息 `Error: connect ECONNREFUSED`

**解决**:
1. 检查 MySQL 服务是否运行
2. 检查数据库配置 (backend/.env)
3. 检查数据库是否存在
```bash
cd d:\相机租赁\backend
node check-database.js
```

#### 问题3：缺少依赖包
**症状**: 错误信息 `Error: Cannot find module 'xxx'`

**解决**:
```bash
cd d:\相机租赁\backend
npm install
```

### 6.2 小程序无法连接后端

#### 问题1：网络错误
**症状**: 提示 "request fail Network Error"

**解决**:
1. 检查后端服务是否运行
2. 检查 Cloudflare Tunnel 是否运行
3. 在微信开发者工具中开启 "不校验合法域名"
   - 设置 → 项目设置 → 本地设置 → 勾选 "不校验合法域名..."

#### 问题2：域名未配置
**症状**: 提示 "不在以下 request 合法域名列表中"

**解决**:
1. 登录微信公众平台
2. 开发 → 开发管理 → 开发设置 → 服务器域名
3. 添加 `https://sunnycamera.online` 到 request 合法域名

### 6.3 登录失败

#### 问题1：数据库表不存在
**症状**: 登录无响应或报错

**解决**:
```bash
cd d:\相机租赁\backend
node setup-db.js
```

#### 问题2：后端服务未运行
**症状**: 提示 "登录失败，请检查网络连接"

**解决**:
1. 检查后端服务是否在运行
2. 检查 Cloudflare Tunnel 是否在运行
3. 检查网络连接
